## การจำแนกการใช้ที่ดินในจังหวัดจันทบุรี
พื้นที่ศึกษา จังหวัดจันทบุรี

## 1. ข้อมูลเบื้องต้นของการทำโมเดล

1.1 การแบ่งที่ดิน
แบ่งเป็น 5 คลาส ได้แก่
1. เกษตร
2. ป่า
3. เมือง
4. น้ำ
5. พื้นที่เบ็ดเตล็ด (อื่นๆ)

1.2 Training Samples

ใช้ข้อมูลจาก ESA World cover

1.3 Train/Validation

มีการทำ 80/20 แบบสุ่ม โดยนักศึกษาสุ่ม 1000 จุด ได้ Train size: 4055 และ Test size: 945

โดยมีข้อดีข้อเสียต่างกันดังนั้น
* แบบสุ่ม 80/20 ทำได้ง่าย ใช้ได้ดีกับข้อมูลทั่วไปและรันเร็ว แต่มีข้อเสียคือ จุดใกล้ๆกันมักมีลักษณะคล้ายกัน (spatial autocorrelation) ทำให้ตัว train กับ test เหมือนกัน
* Spatial Cross-validation มีข้อดีคือ ข้อมูลมีความเป็นจริงมากกว่า ลดปัญหาspatial autocorrelation แต่มีข้อเสียคือ ทำค่อนข้างยสากกว่าแบบสุ่ม ต้องมีการแบ่งพื้นที่ให้ดี ค่า Accuracy จะต่ำลง

1.4 Feature ที่ใช้

1. B2 (Blue) ตรวจจับน้ำ ใช้แยกน้ำกับพื้นดิน
2. B3 (Green) ช่วยตรวจจับพืชพรรณและคุณภาพน้ำ
3. B4 (Red) ตรวจจับ chlorophyll ในพืช
4. B8 (NIR) ใช้สำหรับตรวจจับและวิเคราะห์ข้อมูลที่ตาเปล่ามองไม่เห็น ช่วยจำแนกสิ่งปลูกสร้าง, ดิน, ป่าไม้ และพื้นที่เกษตรออกจากกันได้ชัดเจนกว่าภาพสีปกติ 
5. B11 (SWIR) ตรวจจับความชื้นในดินและพืช มักใช้ดูพื้นที่แห้ง
6. NDVI (Normalized Difference Vegetation Index) ใช้วัดพื้นที่ที่มีพืช
7. NDBI (Normalized Difference Built-up Index) ใช้ตรวจจับพื้นที่เมือง/สิ่งปลูกสร้าง
8. NDWI (Normalized Difference Water Index) ใช้ตรวจจับน้ำหรือความชื้นบนพื้นผิว

## 2. การเปรียบเทียบอัลกอริทึ่ม
ในที่นี้ใช้เป็น Random Forest และ Gradient Tree Boost

2.1 Random Forest — ปรับ numberOfTrees และสังเกตผลต่าง

นักศึกษาเลือกใช้เป็น RF 100 ต้นและ 300 ต้น ผลที่ได้คือ มีค่า Overall Accuracy ต่างกันเล็กน้อย โดยในการเทรน Random Forest แบบใช้ 100 ต้น ได้ค่า OA อยู่ที่ 0.715 ในขณะที่แบบ 300 ต้น มีค่า OA อยู่ที่ 0.719 แสดงให้เห็นว่าตัว RF เรียนรู้ได้เต็มที่แล้วตั้งแต่ 100 ต้น

2.2 การเปรียบเทียบอัลกอริทึ่ม Random Forest vs Gradient Tree Boost

จากการเปรียบเทียบประสิทธิภาพของโมเดล Random Forest และ Gradient Tree Boost โดยพิจารณาทั้ง Overall Accuracy, Kappa รวมถึงค่า F1-score, Producer’s Accuracy (PA) และ User’s Accuracy (UA) พบว่า Random Forest ให้ผลลัพธ์โดยรวมดีกว่าเล็กน้อย เมื่อพิจารณาในระดับรายคลาส ค่า F1-score ของ Random Forest มีแนวโน้มสูงกว่าในคลาสหลัก สะท้อนถึงความสมดุลที่ดีกว่าระหว่าง precision (UA) และ recall (PA) ขณะที่ Gradient Tree Boost แม้จะสามารถจับ pattern บางคลาสได้ดี แต่มีค่า F1-score โดยรวมต่ำกว่าเล็กน้อย จึงสามารถบอกได้ว่า Random Forest อาจจะ generalize กับข้อมูลได้ดีกว่า ในขณะที่ Gradient Tree Boost อาจยังไม่ได้รับการปรับจูนพารามิเตอร์อย่างเหมาะสม หรือข้อมูลที่ใช้ยังไม่ซับซ้อนเพียงพอให้โมเดลแบบ boosting แสดงศักยภาพได้เต็มที่

## 3. วิเคราะห์ Feature Importance

3.1 Band หรือ Index ใดมีความสำคัญมากที่สุด? — สอดคล้องกับความเป็นจริงทางกายภาพหรือไม่?
- จากการทำ Feature Importance ของ Random Forest แบนด์ที่สำคัญที่สุดคือ B11 (SWIR) ส่วน Index ที่สำคัญที่สุดคือ NDBI เนื่องจากตัว feature ทั้งสองนี้นี้ช่วยโมเดลแยกพื้นที่ urban vs non-urban ได้ง่าย ทำให้โมเดลจำแนก class ได้แม่นยำ จึงทำให้มีค่า feture importance ที่สำคัญที่สุด
- ส่วนของ Gradient Tree Boost พบว่า B11 สำคัญที่สุด ส่วน NDVI, NDWI และ NDBI ก็มีความสำคัญสูง การตัด features ที่มีความสำคัญต่ำ เช่น B3 ออก อาจลดความแม่นยำเล็กน้อย แต่โมเดลยังสามารถจำแนก class ได้ดีและ train เร็วขึ้น

3.2 ถ้าตัด Feature ที่มีความสำคัญต่ำออก ความแม่นยำเปลี่ยนไปอย่างไร?

จากการทำ Feature Importance อย่างที่บอกไปว่าตัว B11 และ NDBIมีความสำคัญมากที่สุด อย่างไรก็ตาม features อื่นๆก็มีค่าความสำคัญใกล้เคียงกัน หากตัด feature ที่มีความสำคัญต่ำออก ในที่นี้เป็น B8 โมเดลอาจมีความแม่นยำลดลงเล็กน้อย การแยกป่าไม้และเกษตรอาจจะมีความผันผวนขึ้นเนื่องจากเป็นตัวหลักให้ NDVI ด้วย
