# ความเสี่ยงไฟป่า (Wildfire Risk)
พื้นที่ศึกษา จังหวัดเชียงใหม่

## การสร้างและรัน Spatial Model

**1. การ Normalize แต่ละ Factor**
- ข้อมูล LST จาก MODIS ได้มีการ - MODIS LST ให้ค่า Kelvin × 0.02 → ต้องแปลงเป็นช่วง 0–1 เพื่อให้สเกลเท่ากัน, ปรับunitScale(min, max) → (value - min) / (max - min) และ clamp(0,1) → กันค่าเกินขอบเขต
- ข้อมูล NDVI 0–0.8 แทนความหนาแน่นของพืชยิ่งเขียว → เชื้อเพลิงเยอะ → เสี่ยงสูง จากนั้นทำการ Normalization ทำให้สเกล 0–1 เพื่อเอามารวม AHP ได้
- NDMI ใช้สูตร.unitScale(-0.5, 0.5).subtract(1).multiply(-1).clamp(0,1) คือ invert และ scale ให้:dryness_norm = 1 - ((NDMI - (-0.5)) / (0.5 - (-0.5))) แปลว่ายิ่งแห้ง → ค่ามาก → เสี่ยงสูง
- ข้อมูลประวัติไฟ นับจำนวนไฟแต่ละ pixel 0–5 ครั้ง จากนั้น Normalized ให้เป็น 0–1 เพื่อเอาไปคูณ weight

2. การกำหนด Weight และ Combine 

- มีการ Normalize แต่ละ Factor → ทำให้สเกลเท่ากัน (0–1)
- จาก Dryness = 0.458 Heat = 0.539 Fuel Density = 0.424 History = 0.412 เป็น Heat มี Mean Risk สูงสุด (0.539) → weight สูงที่สุด (0.30) History มี Mean Risk ต่ำสุด (0.412) → weight น้อยที่สุด (0.22) Dryness & Fuel Density อยู่กลาง ๆ → weight 0.25 & 0.23
- ได้ weight sum ออกมาเท่ากับ Fire Risk=0.25⋅dryness_norm+0.30⋅lst_norm+0.23⋅fuel_density_norm+0.22⋅history_norm

3. การแบ่งระดับความเสี่ยง/ความเหมาะสม (Thresholds) — มาจากไหน?

Risk ≤ 0.30 → Class 1 → Low Risk (Green)
0.30 < Risk ≤ 0.40 → Class 2 → Medium Risk (Orange)
Risk > 0.40 → Class 3 → High Risk (Red)

เนื่องจาก Risk Score ส่วนใหญ่มีในช่วง 0.2–0.45 การตั้ง Thresholds ที่สูงไปทำให้แผนที่ Risk map ออกมาว่างเปล่าได้ 

## Sensitivity Analysis

1. ลองเปลี่ยนน้ำหนักของปัจจัยที่สำคัญที่สุด ±20% แล้วดูว่าผลลัพธ์เปลี่ยนอย่างไร

ค้นพบว่าพื้นที่จุด risk score เป็น robust ทั้งหมดแม้จะเพิ่มหรือลดน้ำหนักของปัจจัยที่มีผลที่สุดก็ตาม

2. พื้นที่ไหนที่โมเดลให้ผลคงที่ (Robust) และพื้นที่ไหนที่ไม่แน่นอน?

จากผลลัพธ์ของโมเดล เมื่อปรับน้ำหนักของปัจจัยที่สำคัญที่สุด (Heat) ±20% พบว่า Risk Score ของพื้นที่ทั้งหมดแทบไม่เปลี่ยน หมายความว่าพื้นที่ที่โมเดลประเมินเป็นเสี่ยงสูงหรือเสี่ยงต่ำอยู่แล้วผลลัพธ์ค่อนข้างคงที่ (Robust) โมเดลสามารถจำแนกพื้นที่เสี่ยงได้ชัดเจนแม้ปรับน้ำหนัก factor ทำให้ไม่มีพื้นที่ที่เปลี่ยนระดับความเสี่ยงตามน้ำหนักหรือที่เรียกว่า Sensitive Area 

3. ข้อสรุปของ Sensitivity Analysis มีผลต่อความเชื่อมั่นในโมเดลอย่างไร?

- พื้นที่ Robust → Risk Score ชัดเจน → โมเดลมั่นใจว่าเป็นพื้นที่เสี่ยงสูงหรือเสี่ยงต่ำจริง
- ถ้าพื้นที่ Sensitive → Risk Score เปลี่ยนตาม weight โมเดลไม่แน่ใจ อาจจะต้องระวังการตัดสินใจ หรืออาจปรับ threshold, เพิ่ม factor ใหม่ได้

## Validation กับข้อมูลจริง

ในที่นี้นักศึกษา validate ข้อมูลความเสี่ยงไฟป่ากับข้อมูล MODIS Fire Burned Area พบว่ามีค่า Accuracy อยู่ที่ 52.03% ซึ่งหมายความว่า จาก pixel ที่ถูกบันทึกว่าเกิดไฟจริงทั้งหมด 2,847 pixel โมเดลสามารถทำนายถูกได้ประมาณ 1,481 pixel แม้ค่า accuracy จะไม่สูง แต่มีข้อเสนอแนะคือ โมเดลนี้เป็นโมเดลที่พิจารณาเฉพาะปัจจัยธรรมชาติ เช่น ความแห้งของเชื้อเพลิง (NDMI), ความหนาแน่นของเชื้อเพลิง (NDVI), ความร้อน (LST) และประวัติการเกิดไฟ (History) โดยไม่ได้รวมปัจจัยจากกิจกรรมของมนุษย์ ทำให้โมเดลอาจจะไม่สามารถใช้ในการทำนายความเสี่ยงไฟป่าได้ 100% แต่สามารถนำไปปรับปรุงเพิ่มปัจจัยได้ในอนาคต

## คำถามสำหรับใช้คิดและตอบใ README
1. ถ้ามีข้อมูล Ground Truth จริง คิดว่า Accuracy ของโมเดลนี้จะอยู่ในระดับใด? เพราะอะไร?

ถ้าเรามีข้อมูล Ground Truth จริงจากการสำรวจภาคสนาม (เช่น การสำรวจ pixel ของไฟป่าแบบละเอียด) คาดว่า Accuracy ของโมเดลน่าจะสูงกว่าปัจจุบัน เพราะจะสามารถประเมินได้อย่างตรงจุดกว่าว่า pixel ไหนเกิดไฟจริงหรือไม่ อย่างไรก็ตาม โมเดลนี้ยังขึ้นอยู่กับปัจจัยธรรมชาติเท่านั้น การเกิดไฟจริงยังมีปัจจัยมนุษย์เข้ามาเกี่ยวข้อง เช่น การเผาที่เกษตรกรรม ดังนั้นแม้ข้อมูล Ground Truth จะมีความแม่นยำสูงแต่โมเดลเองก็ยังมีข้อจำกัดจากปัจจัยที่ไม่ได้รวมอยู่ ทำให้ Accuracy อาจปรับขึ้น แต่ไม่ถึงระดับใกล้ 100%

2. ปัจจัยที่ไม่ได้ใส่ในโมเดลแต่น่าจะสำคัญมีอะไรบ้าง? ทำไมถึงไม่ใส่?

กิจกรรมของมนุษย์ เช่น การเผาที่เกษตรกรรม, การทิ้งก้นบุหรี่, การสร้างไฟตั้งแคมป์
ข้อมูลสภาพอากาศ เช่น ลม, ฝนตกเฉียบพลัน, ความชื้นสัมพัทธ์
การจัดการพื้นที่ป่า เช่น การตัดไม้, การปลูกพืช, การสร้างแนวกันไฟ

เหตุผลที่ไม่ใส่เนื่องจาก ข้อมูลที่กล่าวมามีความซับซ้อน,ไม่มี dataset ครอบคลุม และหาค่อนข้างยากมาก

3. โมเดลนี้ใช้ได้ดีในระดับ Scale ใด (ตำบล / อำเภอ / จังหวัด) เพราะอะไร?

ใช้ได้ดีในระดับจังหวัดหรืออำเภอ เพราะว่า Input เป็น MODIS, Landsat, FIRMS ซึ่ง spatial resolution ประมาณ 30–1000 เมตร ข้อมูลไม่ได้ละเอียดขนาดระดับตำบล เป็นการดูพื้นที่ความเสี่ยงแบบกว้างมากกว่า

4. ถ้าต้องนำโมเดลนี้ไปอัปเดตปีละครั้ง ขั้นตอนใดที่ต้องทำซ้ำและขั้นตอนใดที่คงที่?

- ขั้นตอนที่ต้องทำซ้ำทุกปี คือ 

1.1 ดึงภาพ Landsat / MODIS / FIRMS ของปีใหม่ 

1.2 คำนวณ NDVI, NDMI, LST, History ของปีล่าสุด

1.3 ทำ Normalization ของแต่ละ factor

1.4 รวม Weighted Sum (Risk Score) และแบ่ง Threshold ใหม่ตาม distribution หรือ pattern ของปีนั้น

- ขั้นตอนที่ต้องคงที่ คือ

2.1 การกำหนด สูตรคำนวณ Risk Score (weighted sum)

2.2 การกำหนด น้ำหนักปัจจัย (AHP weights) 

2.3 การตั้ง สีและระดับ Risk Level (Low/Medium/High)
